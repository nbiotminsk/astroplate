# Инструкция по интеграции E-POS API «Хуткi Грош» для ИИ агентов

Данный документ предназначен для ИИ агентов и разработчиков, реализующих интеграцию с сервисом оплаты "Хутки Грош" (Hutki Grosh) для выставления счетов в ЕРИП/E-POS.

## Базовая информация

- **Протокол:** REST API (HTTP GET, POST, DELETE).
- **Формат данных:** JSON (указывать `Content-Type: application/json` и `Accept: application/json`).
- **Кодировка:** UTF-8.
- **Базовый URL (Production):** `https://www.hutkigrosh.by/API/v1`
- **Базовый URL (Sandbox):** `https://trial.hgrosh.by/API/v1`
- **Аутентификация:** Cookie-based. После метода `LogIn` сервис возвращает cookies, которые необходимо передавать в последующих запросах. В конце работы вызывается `LogOut`.
- **Формат дат:** JSON .NET нотация: `\/Date(мсек_от_1970+0300)\/`. Пример: `\/Date(1309381200000+0300)\/`.

## Основные этапы (Workflow) для выставления счета

### 1. Аутентификация (`/Security/LogIn`)

- **Метод:** POST
- **URL:** `https://www.hutkigrosh.by/API/v1/Security/LogIn`
- **Тело запроса:**

```json
{
  "user": "username@org.com",
  "pwd": "password"
}
```

- **Ответ:** `true` при успехе, `false` при ошибке авторизации. ОБЯЗАТЕЛЬНО: Сохранить полученные параметры `Set-Cookie` из заголовков ответа.

### 2. Выставление счета (`/Invoicing/Bill`)

- **Метод:** POST
- **URL:** `https://www.hutkigrosh.by/API/v1/Invoicing/Bill`
- **Тело запроса:**

```json
{
  "billID": null,
  "eripId": 40000001, // Идентификатор услуги E-POS/ЕРИП (выдается менеджером)
  "invId": "ORDER-1234", // Номер заказа в системе магазина
  "dueDt": "\/Date(1683417600000+0300)\/", // Срок действия счета (в будущем)
  "addedDt": "\/Date(1683158400000+0300)\/", // Дата выставления счета
  "payedDt": null,
  "fullName": "Иванов Иван Иванович", // ФИО плательщика
  "mobilePhone": "+375291234567",
  "notifyByMobilePhone": false, // Отправлять ли SMS
  "email": "client@example.com",
  "notifyByEMail": false,
  "fullAddress": "г.Минск, ул. Ленина, д.1",
  "amt": 50.5, // Общая сумма
  "curr": "BYN", // Валюта (обычно BYN)
  "statusEnum": 0, // Статус: 0 или NotSet
  "info": "Оплата заказа ORDER-1234", // Инфо о заказе
  "products": [
    {
      "invItemId": "ITEM-1",
      "desc": "Товар 1",
      "count": 1,
      "amt": 50.5
    }
  ]
}
```

- **Ответ:**

```json
{
  "status": 0,
  "billID": 4000000528202700 // Внутренний ID выставленного счета
}
```

- Внимание: `status` равный `0` означает успех. Любое другое значение - код ошибки.

### 3. Получение QR-кода счета (опционально) (`/Invoicing/BillQRCode`)

Для E-POS часто нужно показать пользователю QR-код для сканирования и оплаты через приложение банка.

- **Метод:** GET
- **URL:** `https://www.hutkigrosh.by/API/v1/Invoicing/BillQRCode({billID},image,300,300)`
- Возвращает QR-код. Для получения Base64 или массива пикселей используйте `{resultType} = text` и формируйте QR сами, либо забирайте PNG.

### 4. Завершение сессии (`/Security/LogOut`)

- **Метод:** POST
- **URL:** `https://www.hutkigrosh.by/API/v1/Security/LogOut`
- **Заголовки:** Передать cookies, полученные при авторизации.
- **Тело запроса:** пусто.

## Особенности реализации на PHP

Библиотека `PECL_HTTP`, упоминаемая в документации API, устарела. Используйте **cURL**.
При использовании `curl`, обращайте внимание на парсинг `Set-Cookie` заголовков после `LogIn` и их склейку в строку для передачи в заголовке `Cookie` при вызове `/Invoicing/Bill` и `/Security/LogOut`.

## Нотификация об оплате (Webhooks/Callbacks)

Когда счет оплачивается, сервис "Хутки Грош" вызывает ваш Callback URL методом GET, добавляя параметр `?purchaseid={billId}`.
Ваш скрипт должен:

1. Вызвать метод GET `/Invoicing/Bill({billId})` на стороне сервера (с предварительным `LogIn`) для проверки `statusEnum` счета (5 = Оплачен).
2. Вернуть HTTP Code `200 OK`.
   Если вернуть другой код, система попытается отправить webhook еще 3 раза.

## Коды ошибок Invoicing (Частые)

- `0 (0x0)` - Успешно
- `3221291523 (0xC0010203)` - Общая ошибка выставления счета (неверный формат даты, будущее время addedDt и т.д.)
- `3221291525 (0xC0010205)` - Счет не уникальный (дубликат)
- `3221291529 (0xC0010209)` - Не указана сумма счета

## Рекомендации для ИИ

- При генерации JSON payload строго соблюдайте типы: сумма (`amt`) должна быть числом (например, `120.00`), `billID` должно быть `null` при создании.
- Внимательно обрабатывайте формат даты `\/Date(...)\/`. В PHP это можно сделать так: `'\/\/Date(' . (time() * 1000) . '+0300)\/\/'`. Обратите внимание на экранирование слешей внутри JSON. Встроенный `json_encode` не экранирует слеши особым образом по умолчанию, нужно убедиться, что строка в итоговом JSON выглядит в точности как `\/Date(1683417600000+0300)\/`.
