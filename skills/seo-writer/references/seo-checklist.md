# Чек-лист SEO (Статьи и Товары)

Перед публикацией или генерацией контента проверьте материал по следующим пунктам:

## Техническое СЕО
- [ ] **ЧПУ URL (slug)** короткий, понятный и на латинице (например: `schetchik-vody-zenner.md`).
- [ ] **Meta title** (SEO-заголовок) содержит главное ключевое слово и не превышает 60 символов.
- [ ] **Meta description** заполнен, содержит УТП или призыв к действию (CTA) и не превышает 160 символов.
- [ ] Заголовок **H1** присутствует в тексте (или генерируется из title) ровно один раз.
- [ ] Подзаголовки (**H2, H3**) структурируют текст логически, содержат LSI-фразы.
- [ ] У всех добавленных изображений прописан атрибут **alt** (в Markdown: `![alt текст](url)`).

## Контент
- [ ] **Объем текста**: 2000–3000 символов (для статей блога) / 1000–2000 символов (для товаров).
- [ ] **Уникальность**: минимум 85% по результатам проверки (например, text.ru).
- [ ] **Плотность ключей**: не более 3% (отсутствует переспам, фразы вписаны естественно).
- [ ] **Читабельность**: текст разбит на короткие абзацы (не более 3-5 предложений).
- [ ] В тексте использованы списки (маркированные или нумерованные) для легкого чтения.
- [ ] Присутствует **внутренняя перелинковка** (минимум 1-2 ссылки на смежные товары, категории или статьи).

## Специфика товаров (Каталог / Store)
- [ ] Корректно заполнены коммерческие поля Frontmatter: `price`, `price_currency`, `category`, `brand`.
- [ ] Заполнено `short_description` (2-3 предложения) для превью карточки.
- [ ] В описание добавлены четкие списки характеристик (`features`) и блок частых вопросов (`faq`).
- [ ] Текст фокусируется на выгодах клиента и технических преимуществах оборудования.

## Микроразметка (JSON-LD)
- [ ] На страницу добавлен или корректно генерируется компонент `<JsonLd />` с верным типом:
  - `BlogPosting` — для статей блога (проверить наличие `date`, `author`, `image`).
  - `Product` — для карточек товаров (проверить наличие рейтинга `review` или `aggregateRating`, чтобы Google отрисовал звездочки в выдаче).
